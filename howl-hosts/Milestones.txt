howl-term-mvp
      1. Host lifecycle hook parity (start/ready/fail) on both hosts.
      2. Shell spawn + PTY wiring through `howl-term` handles on both hosts.
      3. Render API wiring MVP on both hosts:
          - host render entry uses `renderFrameSized(render_w, render_h, grid_w, grid_h, texture)`
          - host reads redraw truth from `dirtySnapshot()`
          - host calls `acknowledgePresented()` only after successful present
          - window layer event wait/drain wakes render path; no host tight polling loop
      4. One-way output proof into vt-core on both hosts.
      5. Font rendering MVP:
          - atlas upload path live in render impls
          - glyph quads rendered from real rasterized glyphs
          - fallback handling for missing glyphs
      6. Input mapping proof (host-native input -> vt-core -> shell echo).
      7. Resize correctness with split sizing contract:
          - grid size drives cell math
          - render size drives viewport/target texture size
          - size/scale math updates only on resize/scale events, not per draw pass
      8. Stability/hygiene pass (warnings/errors only, no healthy-path logs, repeated run checks on both hosts).

howl-renderer-impl
      1. Contract lock:
          - `howl-term` owns redraw truth (`dirty: none|partial|full`, damage bounds, row spans)
          - hosts own wake/schedule/present only
          - dirty clears only after successful present-ack
      2. `Term` structure completion:
          - explicit render-facing snapshot state on `Term` (grid, cursor, colors/attrs, dirty metadata)
          - stable API surface: `renderFrameSized(...)`, `dirtySnapshot()`, `acknowledgePresented()`
          - resize path sets full dirty and rebuilds render-facing state deterministically
      3. render-impl structure completion:
          - renderer consumes term snapshot + dirty metadata, no host-side dirty inference
          - full path redraws complete frame; partial path redraws only damaged rows/spans
          - target texture + viewport sizing obeys render size, grid math obeys grid size
      4. Host integration completion (SDL/GLFW + Android together):
          - event-driven wake model (no unconditional tight render loop)
          - window layer drains event batches and publishes normalized signals into core pipeline
          - one render+present pass per coalesced wake batch
          - `acknowledgePresented()` only on successful present
      5. Verification gate:
          - same terminal-visible behavior across SDL host and Android host for dirty/full/partial scenarios
          - scale/resize/input/blink flows proven with contract logs/tests

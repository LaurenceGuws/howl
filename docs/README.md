# Parent Workflow Docs

This directory contains parent-level workflow material for cross-repo execution.
Durable product authority stays in `../architecture_docs/authority/`.

## Layout

- `architect/` — parent workflow, milestone progress, and reviews.
- `engineer/` — active queues and report checklists for execution handoff.

## Rule

Use this directory for process state, sprint planning, reviews, queues, and
handover requirements. Do not put durable module ownership or dependency
authority here; that belongs in `architecture_docs/authority/`.
